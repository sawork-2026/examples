# bookstore-stateful

这个示例基于前面 MVC 章节里的书店应用改造而来，并额外加入了一个最小可运行的**服务端状态管理**实现。

新增能力：

- 基于 Cookie + 服务端内存字典的 `SessionManager`
- `Request.session()` / `Request.cookies()` 访问能力
- 购物车状态保存在服务器上
- 结账流程步骤保存在服务器上
- 可直接查看当前会话在服务器端保存的状态快照

快速体验：

1. 进入这个示例目录
2. 安装依赖：`pip install -r requirements.txt`
3. 启动服务：`python3 main.py 8080`
4. 打开书籍页面，把书加入购物车
5. 继续进入购物车和结账页面，观察服务端保存的会话状态
6. 再访问会话快照接口，直接查看服务器端状态

这个示例的重点不是“RESTful”，而是故意展示一种**服务端保存应用状态**的做法，方便和课件中 REST 的无状态约束形成对比。
